## 📌 Project Overview
This repository contains my solutions for the **42 Lisboa Piscine (January - February 2026)**. It serves as a comprehensive record of my journey through the fundamentals of Shell scripting and C programming. 

The goal of this Piscine is to master low-level logic, memory management, and algorithmic thinking without the use of standard libraries (except where permitted).

## 🛠️ Philosophy: Modular & Clean Code
Unlike many Piscine approaches where logic is often crammed into the `main` function, I prioritize **Separation of Concerns**.
- **Modularity:** Logic is broken down into small, reusable helper functions.
- **Readability:** The `main` function acts as the "orchestrator," making the high-level intent of the program immediately clear.
- **Maintainability:** Code is written following the 42 Norm, ensuring it is robust and easy to debug.

## 💡 How to Use This Repo
If you are a future student looking through these exercises:
1. **Don't just copy.** Try to understand the logic first. You will be caught during your peer evaluation or lifeguard random evaluations. You will also fail your final exam.
2. **Check the Norm.** All `.c` files should be run through `norminette`.
3. **Test thoroughly.** Use your own `main.c` files to test edge cases. I've added my own for the test_explanation.c files, however play around with them. Add more tests, explore.
